# Cybersecurity Frameworks

This document lists the cybersecurity frameworks that are conceptually aligned with the project’s risk management and compliance features.

## NIST Cybersecurity Framework (CSF)
- **Use:** Provides a structure for managing cybersecurity risk across Identify, Protect, Detect, Respond, and Recover.
- **Project relevance:** The app modules for risk assessment, compliance tracking, threat monitoring, and remediation map to NIST CSF functions.

## ISO/IEC 27001
- **Use:** Defines requirements for an information security management system (ISMS).
- **Project relevance:** Vendor management, compliance certification, and documented security controls support ISO 27001 control and audit readiness.

## CIS Controls
- **Use:** Offers prioritized actions to improve cybersecurity posture.
- **Project relevance:** Remediation Tracker and threat monitoring are consistent with CIS Controls such as asset inventory, vulnerability management, and incident response.

## MITRE ATT&CK
- **Use:** Provides a knowledge base of adversary tactics, techniques, and procedures.
- **Project relevance:** Threat & Monitoring Center concepts align with detection and response planning using threat patterns and attacker behavior.

## SOC 2 / Trust Services Criteria
- **Use:** Defines security, availability, processing integrity, confidentiality, and privacy requirements for service organizations.
- **Project relevance:** Compliance & Certification tracking supports audit evidence and reporting expected by SOC 2 assessments.

## PCI DSS (optional)
- **Use:** Specifies security requirements for organizations that handle payment card data.
- **Project relevance:** If vendor data or compliance controls include sensitive payment information, the compliance module can be extended to support PCI DSS items.

## Summary
The file now contains only cybersecurity frameworks and their practical use in the project, without frontend or backend libraries.
